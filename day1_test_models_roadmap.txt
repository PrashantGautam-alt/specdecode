================================================================
STEP 1 — IMPORTS (already done)
================================================================

    import time
    from src.models import ModelLoader

WHY:
- "time" is Python's built-in clock. time.time() returns seconds
  since 1970. Two readings subtracted = elapsed time. That is our
  stopwatch for measuring generation speed.
- "from src.models import ModelLoader": dots = folder path.
  src/models.py is the module; ModelLoader is the class inside it.
  Works only when you run from ~/specdecode, because Python looks
  for the src/ folder relative to where the script lives.
- You do NOT need AutoTokenizer here. Your ModelLoader.load()
  already loads the tokenizer and stores it on the object. That
  was the whole point of writing the class: one clean place that
  handles loading.


================================================================
STEP 2 — LOAD MODEL + TOKENIZER THROUGH YOUR CLASS
================================================================

    loader = ModelLoader("meta-llama/Llama-3.2-1B")
    loader.load()

    model = loader.model
    tokenizer = loader.tokenizer

WHY:
- Line 1 creates the object. Your __init__ only RECORDS the name
  and picks the device ("cuda" because the A5000 is there).
  Nothing heavy happens yet.
- Line 2 does the real work: downloads ~2.5 GB of weights
  (first run only -- cached afterwards), loads them in float16
  (2 bytes/parameter instead of 4 -> half the VRAM), places them
  on the GPU, and calls model.eval() (inference mode, dropout off).
- Lines 3-4: after load(), the model and tokenizer live on the
  loader as attributes. You reach in and take them.

NOTE: the download uses PASSPOLI's internet (IITB's fast line),
not your home connection. Your SSH session only carries text.


================================================================
STEP 3 — TOKENIZE THE PROMPT
================================================================

    prompt = "The capital of France is"
    inputs = tokenizer(prompt, return_tensors="pt").to("cuda")

WHY:
- The model never sees text. It sees integer token IDs -- indices
  into its 128,256-entry vocabulary of word fragments.
  The tokenizer is the codebook between the two worlds.
- return_tensors="pt" -> give me a PyTorch tensor, not a plain
  Python list. The model only eats tensors.
- .to("cuda") -> move the tensor to the GPU. The model's weights
  are on the GPU; both operands of any matrix multiply must live
  on the same device. Forget this and you get:
  "Expected all tensors to be on the same device".
- "inputs" is a small dictionary: input_ids (the token IDs) and
  attention_mask (a helper we will study later -- it marks which
  positions are real tokens).


================================================================
STEP 4 — GENERATE 20 TOKENS, TIMED
================================================================

    start_time = time.time()
    output_ids = model.generate(**inputs, max_new_tokens=20)
    elapsed = time.time() - start_time

WHY:
- model.generate() is HuggingFace's built-in autoregressive loop:
  predict next token, append it, repeat 20 times. TODAY it is a
  black box. TOMORROW (Day 2) you write this loop yourself --
  that is the core of the whole project.
- **inputs unpacks the dictionary into named arguments. It is
  shorthand for: model.generate(input_ids=..., attention_mask=...).
- max_new_tokens=20 -> stop after 20 NEW tokens (prompt excluded).
- The two time.time() calls bracket only the generation, not the
  model loading. We measure what we care about.


================================================================
STEP 5 — DECODE AND PRINT
================================================================

    text = tokenizer.decode(output_ids[0])
    print(text)
    print(f"Generated 20 tokens in {elapsed:.2f} seconds")

WHY:
- generate() returns a BATCH: a 2-D grid of token IDs, one row
  per prompt. We sent one prompt, so we take row 0. That is the
  [0].
- decode() = reverse codebook lookup: IDs -> fragments -> glued
  back into a readable string.
- The printed text contains your PROMPT PLUS the 20 new tokens.
  generate() returns the whole sequence, not just the new part.
- f"..." is an f-string: {elapsed:.2f} substitutes the variable,
  formatted to 2 decimal places.
